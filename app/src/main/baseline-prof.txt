# Hand-authored startup baseline profile for ScanForge.
# profileinstaller ships this with the APK and the platform AOT-compiles these classes/methods on
# first run, cutting cold-start jank on the launch → onboarding/home path. Lines use the standard
# human-readable profile format (HSPL = Hot/Startup/Post-startup; L = include the whole class).
#
# Scope: the app's startup graph (Application, Activity, theme, navigation, first screens). Library
# hot paths (Compose, Hilt, Room) are covered by their own bundled profiles, so we keep ours to the
# app code that actually runs before the first frame.
